# 概念

## 引言

该怎么把Trie用二叉树来实现？一个自然的想法就是用二进制，0为左，1为右。这样，沿着一个字符串的二进制表示就能确定一条路径，到达叶子节点，就是确定了一个字符串。但怎样做才是最高效的，这其实会引发不少问题，Wapiti项目中用了一个数据结构叫CritbitTrie，这里来详细分析下。

## 传统二进制Trie的问题

让我们先看看传统的二进制Trie是怎样的。以字符串 "a", "b", "c", "d" 为例：

```
字符二进制表示:
'a' = 01100001 (97)
'b' = 01100010 (98)  
'c' = 01100011 (99)
'd' = 01100100 (100)
```

按照每层固定检查一个bit位（从bit7到bit0）的规则，构建的完整二进制树如下：

```
                            root
                           /
                      bit7=0
                         |
                       node1
                         \
                      bit6=1
                         |
                       node2
                         \
                      bit5=1
                         |
                       node3
                        /
                   bit4=0
                      |
                    node4
                     /
                bit3=0
                   |
                 node5
                 /   \
            bit2=0   bit2=1
               |       |
             node6    'd'
             /   \
        bit1=0  bit1=1
           |       |
          'a'    node7
                 /   \
            bit0=0  bit0=1
               |       |
              'b'     'c'
```

这样的二叉树虽然具有Trie的全部特性（比如遍历子树得到同前缀字符串），但最大的缺点是**太费存储空间**——需要8层深度，大量无用的单分支节点。

## CritbitTrie的核心思想

CritbitTrie通过**压缩无用层级**来提升存储效率。理解CritbitTrie的关键是明确什么是**Critbit**（关键位）。

### Critbit的定义

**Critbit就是把一组字符串中相同的前缀二进制位去掉后的第一个不同的二进制位。**

以"a", "b", "c", "d"为例：
- 二进制表示：01100001, 01100010, 01100011, 01100100
- 相同前缀：01100（前5位相同）  
- 第一个不同位：**第2位**（从右数第2位）

这个Critbit可以把四个字符分成两组：
- bit2=0: {a, b, c}
- bit2=1: {d}

继续递归分析：
- {a, b, c}的Critbit是第1位，分成 {a} 和 {b, c}
- {b, c}的Critbit是第0位，分成 {b} 和 {c}

最终得到压缩后的CritbitTrie：

```
         (pos=0,bit=2)
         /           \
    bit2=0         bit2=1
       |             |
  (pos=0,bit=1)     'd'
   /         \
bit1=0     bit1=1
  |           |
 'a'    (pos=0,bit=0)
         /         \
    bit0=0       bit0=1
       |           |
      'b'         'c'
```

显而易见，这个树比传统二进制树要紧凑得多，存储效率大幅提升。

## 扩展到变长字符串

### 字典序的理解

对于字符串比较，字典序规则是：
1. 从左到右逐个字符比较
2. 字符更小的排在前面
3. **前缀相等时，短字符串排在前面**（如 "a" < "an"）

### 统一长度策略

为了处理变长字符串，CritbitTrie采用**统一长度策略**：将所有字符串用`\0`填充到最长字符串的长度。

例如，对于字符串集合 {'a', 'b', 'c', 'd', 'aa', 'ab', 'abc'}：

| 原字符串 | 统一长度表示 | 说明 |
|----------|--------------|------|
| "a"      | "a\0\0"      | 用2个\0填充 |
| "b"      | "b\0\0"      | 用2个\0填充 |
| "c"      | "c\0\0"      | 用2个\0填充 |
| "d"      | "d\0\0"      | 用2个\0填充 |
| "aa"     | "aa\0"       | 用1个\0填充 |
| "ab"     | "ab\0"       | 用1个\0填充 |
| "abc"    | "abc"        | 无需填充 |

### 二进制表示分析

bit位置标注: `76543210` (从右到左)

| 字符串 | pos=0 | pos=1 | pos=2 |
|--------|-------|-------|-------|
| "a\0\0" | a(97)=`01100001` | \0(0)=`00000000` | \0(0)=`00000000` |
| "b\0\0" | b(98)=`01100010` | \0(0)=`00000000` | \0(0)=`00000000` |
| "c\0\0" | c(99)=`01100011` | \0(0)=`00000000` | \0(0)=`00000000` |
| "d\0\0" | d(100)=`01100100` | \0(0)=`00000000` | \0(0)=`00000000` |
| "aa\0" | a(97)=`01100001` | a(97)=`01100001` | \0(0)=`00000000` |
| "ab\0" | a(97)=`01100001` | b(98)=`01100010` | \0(0)=`00000000` |
| "abc" | a(97)=`01100001` | b(98)=`01100010` | c(99)=`01100011` |

### 关键bit分析

#### 第1个关键bit：分析整个集合

**集合**：{a\0\0, b\0\0, c\0\0, d\0\0, aa\0, ab\0, abc}  
**分析位置**：pos=0

各字符串在pos=0的字符：
- a\0\0, aa\0, ab\0, abc: 'a'(97) = 01100001
- b\0\0: 'b'(98) = 01100010  
- c\0\0: 'c'(99) = 01100011
- d\0\0: 'd'(100) = 01100100

**bit位分组分析**：
- bit7~bit3: 全部相同，无分组作用
- **bit2**: 0组={a,b,c,aa,ab,abc}, 1组={d} ← **第一个有效分组！**

**结果**：第1个关键bit = **(pos=0, bit=2)**

#### 第2个关键bit：分析左子树

**集合**：{a\0\0, b\0\0, c\0\0, aa\0, ab\0, abc} (bit2=0的组)  
**分析位置**：pos=0

**bit位分组分析**：
- **bit1**: 0组={a,aa,ab,abc}, 1组={b,c} ← **第一个有效分组！**

**结果**：第2个关键bit = **(pos=0, bit=1)**

#### 第3个关键bit：分析{a,aa,ab,abc}子集

**集合**：{a\0\0, aa\0, ab\0, abc} (pos=0,bit1=0的组)  
**分析位置**：pos=1 (因为pos=0都是'a'，无法区分)

各字符串在pos=1的字符：
- a\0\0: '\0'(0) = 00000000
- aa\0: 'a'(97) = 01100001
- ab\0: 'b'(98) = 01100010  
- abc: 'b'(98) = 01100010

**bit位分组分析**：
- **bit6**: 0组={a}, 1组={aa,ab,abc} ← **第一个有效分组！**

**结果**：第3个关键bit = **(pos=1, bit=6)**

#### 继续分析其他子集

按照相同原理，可以得到：
- 第4个关键bit = (pos=1, bit=1) - 区分{aa\0} vs {ab\0, abc}
- 第5个关键bit = (pos=2, bit=6) - 区分{ab\0} vs {abc}  
- 第6个关键bit = (pos=0, bit=0) - 区分{b\0\0} vs {c\0\0}

### 最终树结构

```
root = Node(pos=0, bit=2)
├─ data[0] = Node(pos=0, bit=1)  ← bit2=0: {a,b,c,aa,ab,abc}
│  ├─ data[0] = Node(pos=1, bit=6)  ← bit1=0: {a,aa,ab,abc}
│  │  ├─ data[0] = Value("a")  ← bit6=0: {a}
│  │  └─ data[1] = Node(pos=1, bit=1)  ← bit6=1: {aa,ab,abc}
│  │     ├─ data[0] = Value("aa")  ← bit1=0: {aa}
│  │     └─ data[1] = Node(pos=2, bit=6)  ← bit1=1: {ab,abc}
│  │        ├─ data[0] = Value("ab")  ← bit6=0: {ab}
│  │        └─ data[1] = Value("abc")  ← bit6=1: {abc}
│  └─ data[1] = Node(pos=0, bit=0)  ← bit1=1: {b,c}
│     ├─ data[0] = Value("b")  ← bit0=0: {b}
│     └─ data[1] = Value("c")  ← bit0=1: {c}
└─ data[1] = Value("d")  ← bit2=1: {d}
```

### 零填充的意义

让我们详细分析一下为什么 `data[0] = Value("a")` 在 `Node(pos=1, bit=6)` 处：

#### 追踪路径

1. **到达节点的路径**：
   ```
   root(pos=0,bit=2) → bit2=0 → Node(pos=0,bit=1) → bit1=0 → Node(pos=1,bit=6)
   ```

2. **该节点处理的字符串集合**：{a\0\0, aa\0, ab\0, abc}

3. **pos=1位置的分析**：

   | 字符串 | pos=1字符 | 二进制 | 第6bit |
   |--------|----------|--------|--------|
   | a\0\0 | \0(0) | 00000000 | 0 |
   | aa\0 | a(97) | 01100001 | 1 |
   | ab\0 | b(98) | 01100010 | 1 |
   | abc | b(98) | 01100010 | 1 |

4. **分组结果**：
   - bit6=0组: {a\0\0} (只有1个字符串)
   - bit6=1组: {aa\0, ab\0, abc} (多个字符串)

5. **存储决策**：
   - `data[0]` ← bit6=0的组，只有一个字符串，直接存储为`Value("a")`
   - `data[1]` ← bit6=1的组，多个字符串，继续分叉

#### 关键理解

- **字符串'a'的统一长度表示**：原始"a" → "a\0\0"
- **pos=1位置是\0**：因为原字符串只有1个字符，pos=1是填充的\0
- **\0的第6bit=0**：\0 = 00000000，所有bit都是0
- **路径规则**：bit=0走data[0]，bit=1走data[1]

这个设计的意义在于：**零填充使得短字符串自然成为长字符串的前缀**，从而能够正确区分"a"和"aa"、"ab"、"abc"等以"a"开头的字符串。

## 插入机制：节点裂变理论

### 核心性质

**字符串集合固定 → 树形状固定**

这是CritbitTrie的一个重要性质：相同的字符串集合总是产生相同的树结构，与插入顺序无关。

### 节点裂变机制

插入新字符串的过程可以理解为**节点裂变**：

1. **找到最相似节点**：按现有规则搜索，找到与新字符串最相似的现有字符串
2. **计算关键分叉点**：通过critbit分析确定分叉位置
3. **节点裂变**：将1个节点裂变成3个节点的子结构

#### 实例：从{a,b,c,d}到{a,b,c,d,e}

**原始树结构** {a,b,c,d}：
```
root = (pos=0, bit=2)
├─ data[0] = (pos=0, bit=1)  ← {a,b,c}
│  ├─ data[0] = 'a'           ← bit1=0
│  └─ data[1] = (pos=0, bit=0) ← bit1=1: {b,c}
│     ├─ data[0] = 'b'        ← bit0=0
│     └─ data[1] = 'c'        ← bit0=1
└─ data[1] = 'd'              ← bit2=1 (目标裂变节点)
```

**字符分析**：
- 'd' = 01100100
- 'e' = 01100101
- critbit('d','e') = bit0 (最低位不同)

**裂变过程**：

裂变前（1个节点）：
```
Value('d')
```

裂变后（3个节点）：
```
    Node(pos=0,bit=0)
    /              \
 Value('d')     Value('e')
```

**最终树结构** {a,b,c,d,e}：
```
root = (pos=0, bit=2)
├─ data[0] = (pos=0, bit=1)  ← {a,b,c}
│  ├─ data[0] = 'a'           ← bit1=0
│  └─ data[1] = (pos=0, bit=0) ← bit1=1: {b,c}
│     ├─ data[0] = 'b'        ← bit0=0
│     └─ data[1] = 'c'        ← bit0=1
└─ data[1] = (pos=0, bit=0)  ← bit2=1: {d,e} (裂变后)
   ├─ data[0] = 'd'           ← bit0=0
   └─ data[1] = 'e'           ← bit0=1
```

### 裂变机制的意义

1. **保持字典序**：裂变过程始终维护字典序关系
2. **最小干预**：每次插入只影响必要的节点
3. **确定性**：相同的字符串集合总是产生相同的树形状
4. **局部最优**：每个分叉点都是当前集合的最优选择
